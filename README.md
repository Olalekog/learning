# Learning Repository

## Overview

This repository is used to organize my daily cloud, AWS, DevOps, and certification learning activities by week and day.

The structure follows a simple format:

```text
learning-repo/
├── Week1/
│   ├── Day1/
│   ├── Day2/
│   ├── Day3/
│   ├── Day4/
│   └── Day5/
│
├── Week2/
│   ├── Day1/
│   ├── Day2/
│   ├── Day3/
│   ├── Day4/
│   └── Day5/
│
└── Week3/
    ├── Day1/
    ├── Day2/
    ├── Day3/
    ├── Day4/
    └── Day5/
```

Each `Week#` folder contains daily learning folders such as `Day1`, `Day2`, `Day3`, and so on.

---

## Purpose of This Repository

The purpose of this repository is to:

- Track daily learning progress.
- Organize learning activities by week and day.
- Document AWS, cloud, and DevOps training notes.
- Store daily reports and Daily Head Start updates.
- Track weekly project work.
- Maintain certification study notes.
- Document hands-on labs and command-line practice.
- Keep a record of blockers, lessons learned, and next steps.

## Daily Folder Contents

Each `Day#` folder can include the following files:

| File Name | Purpose |
|---|---|
| `daily-head-start.md` | Documents what was done yesterday, today's plan, and blockers. |
| `end-of-day-report.md` | Documents completed activities, takeaways, pending items, and blockers. |
| `study-notes.md` | Contains notes from self-study, training, or certification preparation. |
| `resources.md` | Stores links, references, commands, and useful learning materials. |
| `commands.md` | Optional file for CLI, Git, AWS CLI, Linux, or WSL commands. |
| `project-notes.md` | Optional file for weekly project tasks and updates. |

## Weekly Project Tracking

Each week may include project-related work inside the appropriate `Week#` folder.

Example:

```text

├── Day1/
├── Day2/
├── Day3/
└── weekly-project.md
```

The weekly project file can include:

- Project title
- Objective
- Tools used
- Architecture or workflow
- Implementation steps
- Challenges
- Lessons learned
- Final outcome
